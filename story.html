01 · ปัญหา

คนตาบอด
อ่านฉลากยาไม่ได้

"ยานี้กินก่อนหรือหลังอาหารดี?"
"นี่ยาแก้ปวด หรือยาแก้ไอกันแน่?"
— คำถามง่ายๆ ที่กลายเป็นเรื่องยากเย็น สำหรับคนที่อ่านฉลากเองไม่ได้

ระบบสวัสดิการของไทยแบ่งผู้พิการเป็น 5 หมวด แต่ละหมวดเจอกำแพงเรื่อง "ฉลากยา" ในรูปแบบที่ต่างกัน:

👁
ทางสายตา
อ่านตัวอักษรไม่ได้เลย ต้องพึ่งคนช่วยอ่านทุกครั้ง
👂
ทางการได้ยิน
ฟังเภสัชอธิบายไม่ได้ ต้องอ่านฉลากเอง — ตัวเล็กเกินไป
🦽
ทางร่างกาย
หยิบฉลากเองยาก ปุ่มเล็กๆ บนแอปยังกดลำบาก
🧠
ทางสติปัญญา
เข้าใจศัพท์การแพทย์ยาก ต้องการคนอธิบายซ้ำๆ
🧓
ผู้สูงอายุ
ตามองไม่ค่อยชัด ขนาดตัวอักษรบนฉลากเล็กเกินไป

เป้าหมาย: เปิดเว็บ ส่องฉลาก ฟังเสียง ถาม AI ต่อ — ฟรี ใช้ในเบราว์เซอร์ ไม่ต้องลงแอป

02 · การสร้าง

8 ขั้น
จากศูนย์ถึงใช้งานได้.

01
DESIGN SCANNER V1

เปิดด้วย dashboard แรก — กล้อง + OCR

เริ่มต้นด้วย Tesseract.js อ่าน OCR ภาษาไทย+อังกฤษ ในเบราว์เซอร์ กล้อง webcam สด มี viewfinder กรอบเหลือง scanline animation ออกแบบสไตล์ "Pharmaceutical Editorial" — Fraunces + IBM Plex Sans Thai + JetBrains Mono

"ไม่ใช้ font generic ไม่ใช้ purple gradient — ต้องดูเป็น production-grade ตั้งแต่นาทีแรก"
02
UPGRADE

เปลี่ยนมาใช้ Gemini Vision — แม่นขึ้น 3 เท่า

Tesseract อ่านฉลากย่นๆ ไม่ค่อยได้ผล จึงเปลี่ยนไปใช้ Google Gemini 2.0 Flash เพราะอ่านลายมือ ฉลากเก่าๆ และตีความบริบทได้ดีกว่ามาก — สกัด 14 fields (ชื่อยา · ขนาด · วิธีใช้ · สรรพคุณ · คำเตือน · วันหมดอายุ ฯลฯ) เป็น JSON

03
QR + NUMBERS

เพิ่ม jsQR + อ่านตัวเลขทุกตัวบนฉลาก

ฉลากยาในไทยมี เลขทะเบียน อย. · barcode · Lot · QR code เพิ่ม jsQR library ถอดรหัส QR ฝั่ง client (ไม่ต้องส่งไป server) และปรับ prompt ให้ Gemini ดึงตัวเลขทุกตัวใส่ใน all_numbers[]

04
UX

รับรูปได้ 4 ทาง — กล้องสด, อัพโหลด, ลากวาง, paste

ผู้สูงอายุบางคนถ่ายรูปฉลากไว้ในมือถือ แล้วอยากแชร์มาทาง LINE เพิ่มความสามารถ upload file · drag & drop · Ctrl+V paste จาก clipboard ใครส่งรูปมายังไงก็เปิดอ่านได้

05
A11Y · CHAT

AI Chat ถามต่อได้ — พูดภาษาไทย ฟังภาษาไทย

อ่านฉลากแล้ว แต่ยังมีคำถามต่อ: "ผลข้างเคียง?" "ทานพร้อมกาแฟได้ไหม?" "ลืมทานทำยังไง?"

เพิ่ม chat panel ใต้ผลลัพธ์ — รักษา context ของยา + multi-turn conversation ที่สำคัญที่สุดสำหรับผู้พิการทางสายตา: กดไมค์พูดภาษาไทยถามได้ (Web Speech API) AI ตอบ + อ่านออกเสียงให้ฟังอัตโนมัติ

มี 8 quick-question chips ให้กดเลย ไม่ต้องคิดเอง — สำคัญมากสำหรับผู้พิการทางสติปัญญา
06
SECURITY

ลบ API key ออกจาก source code

ตอนพัฒนามี DEFAULT_API_KEY ฮาร์ดโค้ดไว้เพื่อทดสอบง่ายๆ แต่ก่อนขึ้น GitHub ต้องลบทิ้งทั้งหมด เพราะ source code public ใครเปิดดูก็เห็น

แก้เป็น: ผู้ใช้ใส่ key เอง → เก็บใน localStorage ของเบราว์เซอร์ตัวเอง เพิ่ม คู่มือ api-key-guide.html สอนขอ key 5 ขั้น พร้อม CSS mockup ของหน้า Google AI Studio

07
PRE-FLIGHT

ตรวจสอบทุกอย่างก่อนขึ้น GitHub

เขียน .gitignore ครอบคลุม — .env · .key · .DS_Store · .claude/ เขียน README.md เต็มรูปแบบ + browser support table เพิ่ม LICENSE MIT พร้อม medical disclaimer ทั้งไทย+อังกฤษ (เพื่อระบุชัดว่า ScanRx ไม่ใช่คำแนะนำทางการแพทย์)

Scan git history ด้วย regex หา AIzaSy[A-Za-z0-9_-]{30,} — สะอาด ไม่มี key หลุด

08

gh repo create --public --push ในคำสั่งเดียว ขึ้น Public repo เปิด GitHub Pages ใช้ branch main — สร้าง index.html เป็น redirect ไปยังแอปหลัก

04 · สิ่งที่เรียนรู้

3 บทเรียน
จากการทำจริง.

① Accessibility ไม่ใช่ feature เสริม

ตอนแรกคิดว่า "อ่านออกเสียง" เป็น nice-to-have แต่พอลองหลับตาใช้แอปจริงๆ พบว่ามันเป็นแกนของผลิตภัณฑ์เลย — ถ้าไม่มี voice readout คนตาบอดใช้ไม่ได้เลยแม้แต่จะรู้ว่า scan สำเร็จหรือไม่

② AI Chat สำคัญพอๆ กับ OCR

ฉลากยาบอกแค่ "วิธีใช้" แต่ผู้ใช้ทั้งหมดอยากรู้มากกว่านั้น — "กินพร้อมกาแฟได้ไหม?" "ลืมทานทำยังไง?" "ตั้งครรภ์ห้ามไหม?" คำถามเหล่านี้ไม่มีบนฉลาก แต่ AI Chat ตอบได้

③ Single-File App ขายของได้

ไม่ต้องมี backend ไม่ต้อง deploy server ไม่ต้องมี database — HTML ไฟล์เดียว เปิดได้ทุกที่ ส่งเป็น attachment ทาง LINE ก็ยังได้ ใช้งานออฟไลน์ก็ได้ (หลังโหลดครั้งแรก) ลด friction ของผู้ใช้ลงเหลือศูนย์
